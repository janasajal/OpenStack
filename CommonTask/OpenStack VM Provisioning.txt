================================================================================
 LAB SCENARIO 01 — OpenStack VM Provisioning
 Topic   : Virtual Machine Provisioning with Flavor, Network,
           Security Group, and Storage Configuration
 
================================================================================

OBJECTIVE
---------
A new virtual machine was required to be provisioned with specific compute
flavor, network attachment, security group, key pair, and additional block
storage. All components were verified before and after provisioning to ensure
the instance was fully operational.

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------
STEP 2 — Instance Creation
--------------------------------------------------------------------------------

The instance was created using all verified parameters in a single command.
The --wait flag was used to hold output until the build process completed.

  $ openstack server create \
      --flavor m1.medium \
      --image RHEL-8.6-x86_64 \
      --network app-internal-net \
      --security-group sg-web-app \
      --key-name app-team-keypair \
      --availability-zone nova:compute-node-01 \
      --wait \
      app-vm-01

  +-----------------------------+--------------------------------------------------+
  | Field                       | Value                                            |
  +-----------------------------+--------------------------------------------------+
  | id                          | vm-1234-5678-abcd-efgh-000000000001              |
  | name                        | app-vm-01                                        |
  | status                      | BUILD                                            |
  | flavor                      | m1.medium                                        |
  | image                       | RHEL-8.6-x86_64                                  |
  | networks                    | app-internal-net=192.168.10.15                   |
  | security_groups             | sg-web-app                                       |
  | OS-EXT-AZ:availability_zone | nova:compute-node-01                             |
  +-----------------------------+--------------------------------------------------+

--------------------------------------------------------------------------------
STEP 3 — Additional Cinder Volume Creation and Attachment
--------------------------------------------------------------------------------

A 100GB block storage volume was created on the ceph-ssd backend and attached
to the instance as a secondary data disk.

The volume was created:

  $ openstack volume create \
      --size 100 \
      --type ceph-ssd \
      --availability-zone nova \
      app-vm-01-datadisk

  +---------------------+--------------------------------------+
  | Field               | Value                                |
  +---------------------+--------------------------------------+
  | id                  | vol-abcd-1234-5678-efgh-000000000001 |
  | name                | app-vm-01-datadisk                   |
  | size                | 100                                  |
  | status              | available                            |
  | type                | ceph-ssd                             |
  +---------------------+--------------------------------------+

The volume was attached to the instance:

  $ openstack server add volume app-vm-01 app-vm-01-datadisk --device /dev/vdb

  Volume app-vm-01-datadisk is being attached to server app-vm-01

--------------------------------------------------------------------------------
STEP 4 — Floating IP Allocation and Association
--------------------------------------------------------------------------------

A floating IP was allocated from the external network pool and associated
with the instance to enable external SSH access.

  $ openstack floating ip create external-net

  +---------------------+--------------------------------------+
  | Field               | Value                                |
  +---------------------+--------------------------------------+
  | floating_ip_address | 10.50.20.101                         |
  | id                  | fip-0001-0001-0001-0001-000000000001 |
  | status              | DOWN                                 |
  +---------------------+--------------------------------------+

  $ openstack server add floating ip app-vm-01 10.50.20.101

The floating IP association was confirmed:

  $ openstack server show app-vm-01 | grep addresses

  | addresses | app-internal-net=192.168.10.15, 10.50.20.101 |

--------------------------------------------------------------------------------

The console log was reviewed to confirm cloud-init completion:

SSH connectivity was tested:

  [cloud-user@app-vm-01 ~]$ hostname

The attached Cinder volume was confirmed visible inside the OS:

  NAME   MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
  vda    252:0    0   40G  0 disk
  └─vda1 252:1    0   40G  0 part /

--------------------------------------------------------------------------------
OUTCOME
--------------------------------------------------------------------------------

The following was confirmed upon completion:

  - Instance app-vm-01 was provisioned in ACTIVE state
  - Flavor m1.medium (4 vCPU / 4GB RAM / 40GB disk) was applied
  - Network interface assigned IP 192.168.10.15 via DHCP
  - Floating IP 10.50.20.101 was associated successfully
  - Cinder volume (100GB ceph-ssd) was attached as /dev/vdb
  - SSH access via key pair was confirmed operational
  - Security group permitted only ports 22, 80, 443
